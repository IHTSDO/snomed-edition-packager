# Usage Guide

Once the application has started, it will be waiting for input commands.

## Package

The `package` command will combine many RF2 packages into one singular RF2 package. This is useful for packaging
an Extension as an Edition, for example, by combining an Extension with International. The table below details the
available parameters.

| Parameter                 | Type    | Default | Example                                                                 | Meaning                                                                                                                                                                                                                                                                                                                                                                                | 
|---------------------------|---------|---------|-------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| shortName                 | String  | *       | ABC                                                                     | The value of this parameter will contribute towards the name of the generated RF2 files. If no value is supplied (i.e. defaulting to `*`) , then the default value of `XX` will be used instead.                                                                                                                                                                                       |
| input                     | Path(s) | *       | /Users/example/Desktop/edition.zip /Users/example/Desktop/extension.zip | The value of this space-separated parameter will determine which locally-stored RF2 packages to combine into one RF2 package. If no value is supplied (i.e. defaulting to `*`), then any RF2 packages in the current working directory will be used instead.                                                                                                                           |
| output                    | String  | *       | MyCombinedPackage                                                       | The value of this parameter will determine the name of the generated RF2 package. If no value is supplied (i.e. defaulting to `*`), then the current time in milliseconds will be used instead.                                                                                                                                                                                        |
| effectiveTime             | String  | *       | 20250101                                                                | The value of this parameter will contribute towards the name of the generated RF2 files. If no value is supplied (i.e. defaulting to `*`), then the current date (`yyyyMMdd`) will be used instead.                                                                                                                                                                                    |
| full                      | Boolean | false   | true                                                                    | The value of this parameter will determine whether to include each given RF2 package's Full sub-folder. If no value is supplied (i.e. defaulting to `false`), then only the Snapshot folder will be processed from each given RF2 package.                                                                                                                                             |
| releasePackageInformation | Path    | *       | /Users/example/Desktop/ReleasePackageInformation.json                   | The value of this parameter will determine whether the `release_package_information.json` file will be copied from the given locally-stored file. If no value is supplied (i.e. defaulting to `*`), then the `release_package_information.json` file will be automatically generated with a best-guess of sensible defaults.                                                           |
| sort                      | Boolean | false   | true                                                                    | The value of this parameter will determine whether to write Components in natural order. If no value is supplied (i.e. defaulting to `false`), then sorting will not happen.                                                                                                                                                                                                           |
| readMe                    | Path    | *       | Users/example/Desktop/readMe.txt                                        | The value of this parameter will determine whether the introductory paragraphs in the `Readme_en_yyymmdd` file will be copied from the given locally-stored file. If no value is supplied (i.e. defaulting to `*`), then the `Readme_en_yyymmdd` file's introductory paragraphs will be automatically generated. Note, the directory listing is not needed in the locally-stored file. |

See below for an example combining all of the above options.

`package --shortName ABC --input /Users/example/Desktop/edition.zip /Users/example/Desktop/extension.zip --output MyCombinedPackage --effectiveTime 20250101 --full true --releasePackageInformation /Users/example/Desktop/ReleasePackageInformation.json --sort true --readMe /Users/example/Desktop/readMe.txt`
